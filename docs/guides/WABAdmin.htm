1     Introduction

This system administration guide is aimed at ArcGIS system administrators are familiar with GIS operations and whose role it is to maintain the GIS system at an administration level. To assist with the administration and configuration of Waahi this guide sets out to describe the custom widgets built specifically for Waahi and the enhanced widget those built on existing out-of-the-box widgets with increased functionality and capability.

After reading this document the system administrator will understand each widgets configuration properties as well as any dependencies on other widgets or exclusions where two widgets cannot co-exist.

3     Waahi Gallery

3.1       Overview

Use the configurable Gallery application to display a collection of content based on what has been shared with you in your organisations Portal for ArcGIS or ArcGIS Online group. The Gallery is a centralised home page for your organisation that makes it easy to find and access apps from your desk or out in the field on a mobile or tablet.

By honouring Portal for ArcGIS or ArcGIS Online authentication settings set for group user content, the Gallery application ensures that you are seeing the right content at the right time.

Secure group content is available after signing into your Portal for ArcGIS or ArcGIS Online account and public group content available by default, as defined by the organisations Portal or ArcGIS Online Administrator. Administrators can set content privacy settings to share each application or content publicly with everyone, within their organisation or with a specific group.

Interact with the Gallery content web page by navigating around the Group categories on the left- hand side, searching, or filtering. In the displayed content you can see the title, description and other metadata before opening the content in a new web page.

3.4     Portal Configuration

Theme settings

         The default configuration settings for the Gallerys colour scheme and text can be sourced from one of two places. By default, the Gallery looks to configuration set in the JSON file (for text and background colours for header, body, and buttons, and logo). I the file is missing or there is more than one of these items then the Gallery will look to the Organisations Portal or ArcGIS Online Shared Theme settings.

         By default, the JSON code for custom theme settings is intentionally left blank enabling the Gallery theme to inherit directly from Portal settings.

Notes

         The Gallery application is supported on all browsers supported by ArcGIS Online, Portal for ArcGIS, and ArcGIS Enterprise. The Gallery is only supported on IE 9 when all content is in ArcGIS Online, Portal for ArcGIS, or ArcGIS Enterprise group, and the gallery application is shared with the public.

         Creating an Application: Applications for use in the Gallery page can be created in Portal under a users or organisations My Content section.

To inherit a logo set in an organisations Portal settings, the headerImage value must be set to True and the headerImageLocation URL value must be left blank in the JSON configuration file for the Gallery page. If the headerImageLocation URL value is left blank or set to false, no logo will appear in the Gallery page header.

Gallery authentication does not currently work with Enterprise users using an external identify provider, such as Azure AD or Active Directory Federation Services.


 

4     Waahi Viewer

Overview

The Waahi Viewer application makes it easy to access, view, query and share geospatial information. The curated collection of widgets and theme form a ready-to-use solution that has well-organised and relevant workflows and processes.

The Viewer is based on the Esri Web AppBuilder framework and benefits from the continuous improvement and investment Esri makes into Web AppBuilder. The Viewer application can inherit from and integrate with your organisations Portal for ArcGIS or ArcGIS Online settings.

Waahi Default Deployment

A Waahi default deployment includes a collection of custom and enhanced Waahi widgets as well as Esri Out-of-the-Box (OOTB) widgets. Some widgets are not part of the default deployment mainly because of the additional custom services that they need. These widgets need to be added specifically during app creation and deployment.

Waahi Widget Configuration

Web AppBuilder Administrators can manage and configure each widget in the widget admin, for example, defining links to organisation data, configuring how data is displayed to the user after a feature query or giving the user more or less options when they are interacting with the widget in the Viewer. All OOTB and Waahi widgets can easily be removed or added in Web AppBuilder (WAB) under the Widget section during configuration.

The sections below will deal with individual Waahi widget and are grouped by the following criteria:

         Custom Waahi Widgets: complete custom widgets not available in WAB in any form. Only Available through Waahi.

         Enhanced Widgets: original Web AppBuilder (WAB) widgets that have been enhanced to increase functionality and capability. These widgets names will be suffixed with Plus to indicate that they are enhanced, for example:

         Additional Esri OOTB widgets: Standard Web AppBuilder widgets

Notes

There are 5 widgets that make up the interrogate framework. These include:

         Identify widget

         Search Plus widget

         Select Plus widget

         Query Plus widget

         Results widget

The Results widget is the foundation of the Interrogate framework. It is a centralised point of display showing all results generated by the other 4 tools in the Interrogate Framework.

4.1     Custom Waahi Widgets

4.1.1    Theme

Overview

Included in a default Waahi deployment: Yes.

The Waahi theme represents the look and feel of the application and is based on the Web AppBuilder Foldable theme style. Content includes a collection of panels, styles, layouts and a set of preconfigured widgets. The application will inherit some organisation configuration from the clients Portal settings.

If Waahi is deployed to your Web AppBuilder, the Waahi Foldable theme will be the default for new apps, and this is the recommended theme for Waahi Widgets. Some functionality may not be supported when using other themes.

Theme colours may be modified under the Style heading, but left default, the theme will inherit from your organisation settings page.

Buttons in all widgets will inherit their style from the Organisation Shared Theme settings under the Button section. If no settings have been set, the buttons will use a default style of grey with white text.

To modify your theme using your organisation settings, as an administrator, go to

Organisation -> Settings -> General

Update the Shared Theme section:

Header Controller widget settings

To change the size of the logo, go to the Header Controller widget settings. For example, you may want your logo to fill the entire height of the header.

The minimum logo height should be 65px by 65px, the maximum should be 90px by 90px.

If the logo is set to full height, you can also set the left margin, typically this is be set to 12px, or 0 for no margin.

Notes

         The header container is a responsive container. The wider the logo and the longer the app title the fewer widgets display in the container. When there are too many widgets for the container, the remainder fall into a More widget to the far right.

         There are some intricacies with the brand colour inheritance. For example, some widgets may have some specificity settings that override the portal inheritance. This is not something that has been planned but is a by-product of the years of code evolution and releases out of Esri. Our aim is to eventually ensure portal inheritance maintains a higher priority over various widget code.

4.1.2    Results widget

Overview

Included in a default Waahi deployment: Yes.

The Results widget is the foundation of the Interrogate Framework. It is a centralised point of display showing all results generated by the other tools in the Interrogate Framework (Search, Identify, Query and Select). It enables the user to view and interact with the attributes of the currently selected feature(s). In addition, it provides administrators with a scalable solution that allows for integration with other systems and software.

This widget can display results for Map Service Layers, Feature Layers and Image Service Layers provided they are configured to display using the Results Panel instead of an on map pop-up.

Included

The product includes the following:

         Displaying the attributes of a map service layer or feature layer.

         Displaying the labels of coded domains and subtypes instead of their assigned code.

         Displaying a customised URL created from a static string and the attributes of the layer.

         It can be customized exactly how each layers attribute data is displayed and which of its related data (if it has any at database level) should show.

         More fields can be added to a table of attributes to display for a feature, such as Hyperlinks and attachments.

Configuration

Setting

Description

Default layer (name or URL)

Set the layer displayed be default in the Result panel (if layer has results).

Fields to exclude

Globally excludes the display of fields by name.

Symbology

Symbol for Results display. Single results can be configured separately to stand out from a larger result set. This includes the colour, transparency, and size of the point.

Edit results display for layers

Customise how each layer is displayed. See the next section for more information.

Zoom Scale

This configuration defines the scale of the map when the user zooms into one feature from the Results panel.

Isolink integration

Waahi can be integrated with Isolink. In this section of the configuration, the WAB Admin sees and can configure the following:

         Enable IsoLink' checkbox

         'IsoLink URL' textbox 

         'Related query names' textbox 

         'Related query order' textbox

         'Enable Mail Merge' checkbox

         'Enable spatial constraints' checkbox

The query names typed into the "Related query names" textbox are the ones for which results will be displayed for in Results widget. Therefore, the WAB Admin decides which IsoLink queries are shown in the app and which are not.

The order in which query names are typed into the "Related query order" textbox is the order in which results will be displayed in Results widget. Therefore, the WAB Admin decides which IsoLink queries are shown in which order.

If no Query name(s) are typed into the "Related query names" textbox, all queries set up in Isolink will be applied.
If no Query name(s) are typed into the "Related query order" textbox, the default order of queries are followed.

Database type queries, as well as spatial Isolink queries can be specified to retrieve results for.

The Results widget has two data display options that can be configured. It can either be configured to show results as it was configured for the web map pop-up (web map pop-up view), or the results display can be custom, overriding the web map pop-up configuration (custom attribute view). Isolink query results will be returned in both cases.

 

Layer configuration web map content view

The result panel content can be configured for each layer in two ways. First, using the web map popup. If a layer has popups configured and enabled, this content will display. Web maps can be configured through your Enterprise Portal or ArcGIS Online.

Layer configuration custom attribute view

Alternatively, custom content can be configured. To configure custom content:

         Add a layer in the Edit results display for layers section

         Check the checkbox to Override web map configuration

         A default list of attributes will be added, these can be refined and edited.

         Select the attributes to display, including for the left and right columns.

         Each attribute can have a display type.

a.       Headings: large text, typically only for the first attribute

b.       Body text: standard display text

c.       Hyperlinks: if the data contains a URL (or a URL can be composed using the data), use this type. The URL content should be added as the left column, while the right column can be an alias or label. For example:

d.       Attachment: enter a heading such as Attachment into the Right column. No URL or field needs to be specified. Any attachments associated with a feature will now be shown in the results panel. Images will show as thumbnails and other file extensions (such as .pdf, .docx etc.) will show as links.

For each attribute, select to display it in the Summary and/or Detailed views.

         Summary attributes are shown for multiple results, typically this will be a shorter display.

         Detailed attributes are shown when clicking through to a single result. Any links should be included here.

If configuring a layer with relationships, the relationships can also be configured in the same way. They will be detected when adding the layer, and each related table can be configured.

Notes

         The Select, Identify, Search widgets must be used in conjunction with the Results Panel widget.

         The order in which layers will be displayed in the Results panel can be configured. If not configured or left blank, by default the Results panel will show the first layer in the dropdown menu for that session and then remember that layer preference for the next results display.

         Symbology is set in the Results Panel and persists across the use of all the interrogate tools identify, search and select.

         To prevent a layer from being displayed in the Results Panel widget by default, the settings can be accessed and changed in the configuration settings for the Identify and Select widgets.

4.1.3    App Switcher widget

Overview

Included in a default Waahi deployment: Yes.

The App Switcher widget allows a user to switch between different web apps. It uses a Portal group, which should contain web mapping apps to do this. The new web app displays to the same extent, provided the spatial references are the same in the two maps. It can either be configured to open the new app in a tab, or in the current tab in the browser.

Configuration

Setting

Description

Portal group name or ID

The name or ID of the group containing apps.

Specify ID instead of name

Set true if you specified a group ID in the settings above, rather than a group name.

Open apps in a new tab

If true, apps will be open in a new browser tab, the existing app will remain open.

Notes

The app switcher does not switch web maps and will not read web maps in your group; it is designed only to open other apps.

4.1.4    Draw and Measure widget

Overview

Included in a default Waahi deployment: Yes.

GBS has created an extended [MK2] Draw widget, called the Draw and Measure widget.

The Draw & Measure widget allows the user to draw graphics and text on the map, see all drawings in a list and edit or manage these graphics. Manage area and distance measurements of graphics by choosing to show or hide the measurements and by changing measurement units. The user can be allowed to choose their own measurement units, or the widget can be configured to change measurement units dynamically based on measurement size.  Distance measurements dynamically update in a textbox next to the cursor as line features are drawn. Measurements for drawn features are displayed, such as area and perimeter for polygons.

Included

The product includes the following:

         Drawing modes (point, line, polyline, freehand polyline, triangle, extent, circle, ellipse, polygon, freehand polygon, buffer, drawing by feature, and text) on the map.

         Editing the shape, colour, and name of graphics added during the session.

         Deleting, moving or rotating graphics added during the session.

         Displaying the length of lines in different units of measurement.

         Displaying the area of a polygon in different units of measurement.

         If configured, the user can change or define the units of measurement that will be used for displaying the length and area of a graphic.

         If configured, the units for measurements will dynamically update based on the size of the drawing.

Configuration

Setting

Description

Include save/open

Allows drawings to be saved to file.

Import service

Service URL, for opening drawings from file. The ExportGraphics service needs to be published to ArcGIS Server. The example below assumes it has been published to CustomUtilities/ExportGraphics, example:

https://secure.gbs.kiwi/arcgis/rest/services/CustomUtilities/ExportGraphics/GPServer/uploads/upload

Shows a list of current drawings to the user.

Point, Line, Polygon, Text and Buffer

Show measurements for each drawing made. This will show length for lines, and areas and segment lengths for polygons.

The label display symbol can be customised.

Allow the user to turn measurement graphics on or off for drawings and to choose the unit in which it displays.

If this checkbox is ticked:

The user can see a checkbox called "Show measurements" under each drawing mode except for Point and Text drawing modes. The user can choose whether drawing measurements will be shown for the drawing by checking or unchecking this checkbox. When ticked, they can also choose in what units the measurement will be.

A list of drawing modes and associated measurement unit options the user can change:

         Point: No Measurements

         Line: Distance Units dropdown menu

         Polyline: Distance Units dropdown menu

         Freehand Polyline: Distance Units dropdown menu

         Triangle: Area Units dropdown menu AND a Distance Units dropdown menu

         Extent: Area Units dropdown menu AND a Distance Units dropdown menu

         Circle: Area Units dropdown menu

         Ellipse: Area Units dropdown menu

         Polygon: Area Units dropdown menu AND a Distance Units dropdown menu

         Freehand Polygon: Area Units dropdown menu

         Text: No measurements

         Buffer: Area Units dropdown menu

         Feature: Area Units dropdown menu

If left unchecked:

The user will not have the option of showing or hiding measurements, nor will the user be able to choose the unit in which measurements are shown. The settings set by the WAB Admin under the Show measurements section in the configuration above will determine whether measurements are added to drawings or not, and in which units.

Distance and Area units

Determines which units are used for measurements, or which are available to the user to choose from.

Notes

The Draw and Measure widget can work with a custom export graphics GP service, published to ArcGIS Server. This is not currently available in the default settings page.

         Edit (reshape, move and/or delete) any of the draw or measure graphics that have added to the map using this widget;

         Access a separate Drawings List tab to view any existing drawings that have been added to the map in the current session. The user can edit, delete, zoom, or rearrange the order of a graphic in this list.  Users can also specify a name for each graphic (excluding text graphics) on the map;

4.1.5    Find Location widget

Overview

Included in a default Waahi deployment: Yes.

The Find Location widget enables users to search for a specific location on a map. The widget provides two search methods for searching locations. Users can search for a location of interest on the map by inputting X and Y coordinates into the fields in the widget and then clicking Locate. Alternatively, they can click on the Identify button, click the mouse cursor on the map and the relative X and Y coordinates will be prepopulated into the widget fields. In this latter Identify example, the coordinates are also overlaid on the map next to the point that was clicked.

The projection systems available in this widget uses what was set up for the Coordinate Plus widget.

Configuration

Setting

Description

Use coordinate projection systems

Set this on, to use the projection systems (or spatial references) set in the Coordinate widget. The same systems will be shown in the drop down. Default: true

Graphic colour

The colour of graphics added to the map displaying coordinates.

Point size

The size of points added to the map.

Font size

The font size of the coordinate label added to the map.

Notes

         Different coordinate systems will require and provide coordinates in different formats. For example, NZTM 2000 uses coordinates in this format: X: 1755313; Y: 5921837 while WGS84 uses coordinates in this format: X:174.742; Y: -36.838.

Before configuring the widget, the ExportData Python geoprocessing service should be published to your ArcGIS Server and configured.

4.2.2    Add Data Plus

Overview

Included in a default Waahi deployment: Yes.

Add data to a map by searching for layers in Portal for ArcGIS or ArcGIS Online content, entering URLs, or uploading local files like shapefiles, CSV, GPX, GeoJSON.

Any graphics added to the map (using the Waahi Draw and Measure widget or the Waahi Find Location widget) will appear above any other added layers, preventing overlapping. https://developers.arcgis.com/web-appbuilder/guide/widget-add-data.htm

4.2.3    Coordinate Plus widget

Overview

Included in a default Waahi deployment: Yes.

The Coordinate widget displays x and y coordinate values on the map. On load of the application, the widget is active and enabled with a pre-configured coordinate system. When a user moves their cursor to different locations on the map, coordinate values change dynamically. Multiple spatial reference systems can be added. There is a set of baseline coordinate systems available for selection to simplify the addition of coordinate systems for commonly used spatial references and enable provision of coordinate system configuration to other widgets.

The coordinate system(s) added to this widget during its configuration will also reflect in the Waahi Find Location widget.

The Coordinate widget has the same configuration options as default Web AppBuilder, see:

https://developers.arcgis.com/web-appbuilder/guide/widget-coordinates.htm

Configuration

Setting

Description

Include widget names and versions

This is typically for development use. When true, shows version numbers for widgets currently in the application.

Content

A rich text editor, allows content to be displayed to users of your app.

Notes

A default coordinate system will be detected, using the web maps basemap. For this to work correctly, after your web map is selected, open and save the Coordinate widget setting.
