2       Gallery

2.1       Overview

Use the configurable Gallery application to display a collection of content based on what has been shared with you in their Portal for ArcGIS or ArcGIS Online group.

By honouring Portal for ArcGIS or ArcGIS Online authentication settings set for group user content, the Gallery application ensures that you are seeing the right content at the right time.

Secure group content is available after signing in to your Portal for ArcGIS or ArcGIS Online account and public group content available by default, as defined by the organisations Portal or ArcGIS Online Administrator. Administrators can set content privacy settings to share each application or content publicly with everyone, within their organisation or with a specific group.

Interact with the Gallery content web page by navigating around the Group categories on the left- hand side, searching, or filtering. In the displayed content you can see the title, description and other metadata before opening the content in a new web page. The Gallery is a centralised home page for your organisation that makes it easy to find and access apps from your desk or out in the field on a mobile or tablet.

    2.4     Portal Configuration

Theme settings

1.       The default configuration settings for the Gallerys colour scheme and text can be sourced from one of two places. By default, the Gallery looks to configuration set in the JSON file (for text and background colours for header, body, and buttons, and logo) and if the file is missing one or more of these items then the Gallery will then look to the Organisations Portal or ArcGIS Online Shared Theme settings.

2.       By default, the JSON code for custom theme settings is intentionally left blank enabling the Gallery theme to inherit directly from Portal settings.

 

2.5       Notes

1.       The Gallery application is supported on all browsers supported by ArcGIS Online, Portal for ArcGIS, and ArcGIS Enterprise. The Gallery is only supported on IE 9 when all content, in the ArcGIS Online, Portal for ArcGIS, or ArcGIS Enterprise group, and the gallery application are shared with the public.

2.       Creating an Application: Applications for use in the Gallery page can be created in Portal under a users or organisations My Content section.

3.       To inherit a logo set in an organizations Portal settings, the headerImage value must be set to True and the headerImageLocation URL value must be left blank In the JSON configuration file for the Gallery page. If the headerImageLocation URL value is left blank or set to false, no logo will appear in the Gallery page header.

4.       Gallery authentication does not currently work with Enterprise users using an external identify provider, such as Azure AD or Active Directory Federation Services.


 

3       Theme

3.1       Overview

The Waahi theme represents the look and feel of the application and is based on the Web AppBuilder Foldable theme style. Content includes a collection of panels, styles, and layouts, and a set of preconfigured widgets. The application will inherit some organisation configuration from the clients Portal settings.

The Waahi Foldable theme is selected by default, and this is the recommended theme for Waahi. Some functionality may not be supported when using other themes.

Theme colours may be modified under the Style heading, but left default, the theme will inherit from your organisation settings page.

To modify your theme using your organisation settings, as an administrator, go to

Organization -> Settings -> General

Update the Shared Theme section.

 

 

3.2       Header Controller widget settings

You may want to change the size your logo is displayed at, this can be done through the Header Controller widget settings. For example, you may want your logo to fill the entire height of the header.

The minimum logo height should be 65px by 65px, the maximum should be 90px by 90px.

If the logo is set to full height, you can also set the left margin, typically this could be set to 12px, or 0 for no margin.

    3.3     Notes

         The header container is a responsive container. The wider the logo and the longer the app title then the less widgets display in the container. When there are too many widgets for the container, the remainder fall into a More widget to the far right.

         There are some intricacies with the brand colour inheritance. E.g. Some widgets may have some specificity settings that override the portal inheritance. This isnt something that has been planned but is a by-product of the years of code evolution and releases out of Esri. Our aim is that eventually we will ensure portal inheritance maintains higher priority over various widget code.

Add data to a map by searching for layers in Portal for ArcGIS or ArcGIS Online content, entering URLs, or uploading local files like shapefiles, CSV, GPX, GeoJSON. 4.5       Coordinate widget

Widget Overview

The Coordinate widget displays x and y coordinate values on the map. On load of the application, the widget is active and enabled with a pre-configured coordinate system. When a user moves their cursor to different locations on the map, coordinate values change dynamically. Multiple spatial reference systems can be added. There is a set of baseline coordinate systems available for selection to simplify the addition of coordinate systems for commonly used spatial references and enable provision of coordinate system configuration to other widgets.

The Coordinate widget has the same configuration options as default Web AppBuilder, see:

https://developers.arcgis.com/web-appbuilder/guide/widget-coordinates.htm

Layer configuration web map content

The result panel content can be configured for each layer in two ways. First, using the web map popup. If a layer has popups configured and enabled, this content will display. Web maps can be configured through your Enterprise Portal or ArcGIS Online.

Layer configuration custom content

Alternatively, custom content can be configured. To configure custom content:

1.       Add a layer in the Edit results display for layers section

2.       A default list of attributes will be added, these can be refined and edited.

3.       Select the attributes to display, including for the left and right columns.

4.       Each attribute can have a display type.

a.       Headings: large text, typically only for the first attribute

b.       Body text: standard display text

c.       Hyperlinks: if the data contains a URL (or a URL can be composed using the data), use this type. The URL content should be added as the left column, while the right column can be an alias or label. For example:

5.       For each attribute, select to display it in the Summary and/or Detailed views.

a.       Summary attributes are shown for multiple results, typically this will be a shorter display.

b.       Detailed attributes are shown when clicking through to a single result. Any links should be included here.

6.       If configuring a layer with relationships, the relationships can also be configured in the same way. They will be detected when adding the layer, and each related table can be configured.

 

Notes

1.       The Select, Identify, Search widgets must be used in conjunction with the Results Panel widget.

1.       The order in which layers will be displayed in the Results panel can be configured. If not configured or left blank, by default, the Results panel will show the first layer in the dropdown menu for that session and then remember that layer preference for the next results display.

2.       Symbology is set in the Results Panel and persists across the use of all the interrogate tools identify, search and select.

3.       To remove a layer from being displayed in the Results Panel widget by default, these settings can be accessed and changed in the configuration settings for the Identify and Select widgets.

If template configured to be layer-dependent, the user must have one feature selected in the configured source layer for this Print Template to be available to the user, e.g. user must have one feature selected in the property layer to generate a set of LIM Maps.

 

If not configured to be layer-dependent, the user will always have the templated options available.

Map Extent

The users current map extent will be used.

The users current map extent will be used.

Layers

All layers currently visible in the map window will be included in the output print file that is generated.

Only the layers that are in the layer files that match the Print Widget configuration will be displayed e.g. If the user chooses to print the LIM Maps, where the Three Wates Print Template is included, only the layers that are in the Stormwater.lyr, Wastewater.lyr and Water.lyr files will be included in the Three Waters print output (regardless of the layers the user currently has visible on the screen).

Draw graphics

Any draw graphics that the user has created in their current session using the print widget will be included in the output print file.

Any draw graphics that the user has created in their current session using the print widget will not be included in the output print file.

Selection graphics

Any selection graphics will be included in the output print file (if they have been added to the map graphics layer).

No selection graphics will be included in the output print file.

 

5.3       Advanced Share widget (Share Plus)

Product Overview

In addition to the functionality of the Share widget, the advanced widget enables users to include the current map extent and the graphics added to a map in a short URL. To add the Share Plus widget, remove the standard Share widget from the app, and add Share Plus.

This widget requires a Share web service (.NET Core) to be installed on a server.
