1       Deploying Waahi to Web AppBuilder for Developers

Once the source code for the Waahi instance has been downloaded, the Waahi deployment app can be used to copy relevant files into your Web AppBuilder (WAB) instance for a typical Waahi setup.

Getting Started

1.1       Download Web AppBuilder for ArcGIS (Developer Edition) SDK

Download the version of WAB required and that is compatible with the latest Waahi release which arent normally far behind the latest version of WAB. The version for which a specific Waahi release has been tested against is in the first part of the release name, for example 2.12.x.x means it was developed and tested against version 2.12 of WAB.

 

Follow this guide to get WAB running on your machine if it hasnt been set up already: https://developers.arcgis.com/web-appbuilder/guide/getstarted.htm

 

If you have an older version of WAB installed and want to download and set up a newer version, download and unzip in parallel to the previous version that was set up (dont replace the previous version). If the new WABs startup.bat file is run, it will ask for the portal URL and an App ID. Use the same app ID that was used for the previous WAB instance. The new WAB version will still run on the same port and URL. One of these WAB instances can now run at a time by running its Startup.bat file and going to the WAB URL in the browser.

1.2       Download and unzip the Waahi application

Once the WAB instance is ready, download and unzip the Waahi-WAB deployment app to a separate folder. For example, you will have the WAB folder with its contents, and the Waahi app with its contents in another:

 
WebAppBuilder-dev
|---- client
|---- docs
|---- server
 
Waahi-WAB
|---- deploy_app
|---- stemapp
|---- node_x64.exe
|---- node_x86.exe
|---- README.md
|---- ui.bat

 

 

 

1.3       Run the deployment app

Run "ui.bat". A browser should open and display the deployment console (on http://localhost:3347/). If you encounter issues check the port setting in the ui.bat file.

Once running:

  • Use the "Existing WebAppBuilder location" input and click set to update the location of your relevant or new WAB install.
  • Select the deployment configuration, typically this will be "Waahi".
  • Use the "Deploy" button to copy relevant files.

If a previous version of Waahi has been deployed previously, files will be overwritten as needed with the version that was downloaded.

A file watcher is also available, which monitors files in the "stemapp" source folder for changes. This is typically used during development only.

1.4       Troubleshooting

If you receive errors, check:

That Web AppBuilder runs by itself, before any Waahi code has been added. This will help pinpoint the problem.

File permissions - Node.js needs to be able to modify files in the Web AppBuilder and Waahi-WAB directories.

1.5       Additional setup

Additional Waahi components that can be optionally installed are:

         Waahi-Services (web services) includes a .NET Core web service for the Share Plus widget, as well as Python geoprocessing services for Print and Data Export widgets.
Waahi-Services repo

         Waahi-Gallery is a minimalistic landing page for your ArcGIS Enterprise or ArcGIS Online users, allowing Waahi and other deployed applications to be easily discovered.
Waahi-Gallery repo

3       Gallery

3.1       Overview

Use the configurable Gallery application to display a collection of content based on what has been shared with you in their Portal for ArcGIS or ArcGIS Online group.

By honouring Portal for ArcGIS or ArcGIS Online authentication settings set for group user content, the Gallery application ensures that you are seeing the right content at the right time.

Secure group content is available after signing into your Portal for ArcGIS or ArcGIS Online account and public group content available by default, as defined by the organisations Portal or ArcGIS Online Administrator. Administrators can set content privacy settings to share each application or content publicly with everyone, within their organisation or with a specific group.

Interact with the Gallery content web page by navigating around the Group categories on the left- hand side, searching, or filtering. In the displayed content you can see the title, description and other metadata before opening the content in a new web page. The Gallery is a centralised home page for your organisation that makes it easy to find and access apps from your desk or out in the field on a mobile or tablet.

    3.4     Portal Configuration

Theme settings

         The default configuration settings for the Gallerys colour scheme and text can be sourced from one of two places. By default, the Gallery looks to configuration set in the JSON file (for text and background colours for header, body, and buttons, and logo) and if the file is missing one or more of these items then the Gallery will then look to the Organisations Portal or ArcGIS Online Shared Theme settings.

By default, the JSON code for custom theme settings is intentionally left blank enabling the Gallery theme to inherit directly from Portal settings.

 

3.5       Notes

         The Gallery application is supported on all browsers supported by ArcGIS Online, Portal for ArcGIS, and ArcGIS Enterprise. The Gallery is only supported on IE 9 when all content, in the ArcGIS Online, Portal for ArcGIS, or ArcGIS Enterprise group, and the gallery application are shared with the public.

         Creating an Application: Applications for use in the Gallery page can be created in Portal under a users or organisations My Content section.

To inherit a logo set in an organizations Portal settings, the headerImage value must be set to True and the headerImageLocation URL value must be left blank In the JSON configuration file for the Gallery page. If the headerImageLocation URL value is left blank or set to false, no logo will appear in the Gallery page header.

Gallery authentication does not currently work with Enterprise users using an external identify provider, such as Azure AD or Active Directory Federation Services.


 

4       Theme

4.1       Overview

The Waahi theme represents the look and feel of the application and is based on the Web AppBuilder Foldable theme style. Content includes a collection of panels, styles, and layouts, and a set of preconfigured widgets. The application will inherit some organisation configuration from the clients Portal settings.

If Waahi is deployed to your Web AppBuilder, the Waahi Foldable theme will be the default for new apps, and this is the recommended theme for Waahi Widgets. Some functionality may not be supported when using other themes.

Theme colours may be modified under the Style heading, but left default, the theme will inherit from your organisation settings page.

Buttons in all widgets will inherit its style from the Organisation Shared Theme settings under the Button section. If no settings have been set, the buttons will use a default style of grey with white text.

To modify your theme using your organisation settings, as an administrator, go to

Organization -> Settings -> General

Update the Shared Theme section:

4.2       Header Controller widget settings

You may want to change the size your logo is displayed at, this can be done through the Header Controller widget settings. For example, you may want your logo to fill the entire height of the header.

The minimum logo height should be 65px by 65px, the maximum should be 90px by 90px.

If the logo is set to full height, you can also set the left margin, typically this could be set to 12px, or 0 for no margin.

    4.3     Notes

         The header container is a responsive container. The wider the logo and the longer the app title then the less widgets display in the container. When there are too many widgets for the container, the remainder fall into a More widget to the far right.

         There are some intricacies with the brand colour inheritance. E.g. Some widgets may have some specificity settings that override the portal inheritance. This isnt something that has been planned but is a by-product of the years of code evolution and releases out of Esri. Our aim is that eventually we will ensure portal inheritance maintains higher priority over various widget code.

5.1       The Waahi Widgets

The following sections describe widgets included in the default Waahi app. Widgets are suffixed with a Plus when they contain custom functionality, to distinguish them from Esris default widgets. You will see the Plus widgets when adding widgets yourself to a controller:

However, once added, they can be labelled according to your preference, and widgets in the startup Waahi application will have a shorter label, such as About.

5.2       About Plus widget

Overview

The Waahi About widget allows administrators to create custom content as a description for your users. Along with this content, the widget will show the attribution and use information required under the creative commons license.

5.3       Add Data Plus widget

Add data to a map by searching for layers in Portal for ArcGIS or ArcGIS Online content, entering URLs, or uploading local files like shapefiles, CSV, GPX, GeoJSON.

Any graphics added to the map (using the Waahi Draw and Measure widget or the Waahi Find Location widget) will appear above any other added layers, preventing overlapping. These graphics layers are temporarily added to (and removed from) the map. Other than this, the Waahi Add Data widget has the same configuration as the default WAB widget, see:

https://developers.arcgis.com/web-appbuilder/guide/widget-add-data.htm

 

5.4       App Switcher Plus widget

Widget Overview

The App Switcher widget allows a user to switch between different web apps. It uses a Portal group, which should contain web mapping apps to do this.

The new web app displays to the same extent, provided the spatial references are the same in the two maps. It can either be configured to open the new app in a tab, or in the current tab in the browser.

Configuration

Setting

Description

Portal group name or ID

The name or ID of the group containing apps.

Specify ID instead of name

Set true if you specified a group ID in the settings above, rather than a group name.

Open apps in a new tab

If true, apps will be open in a new browser tab, the existing app will remain open.

Notes

         The app switcher does not switch web maps and wont read web maps in your group; it is designed only to open other apps.

5.5       Basemap Gallery

The Basemap Gallery widget presents a gallery of basemaps and allows a user to select one from the gallery as the basemap for the app. The basemap can be user-defined or from the organisation or Portal.

The Waahi Basemap Gallery has the same configuration as the default Web AppBuilder widget, information can be found here:

https://developers.arcgis.com/web-appbuilder/guide/widget-basemap.htm

 

 

5.6       Coordinate Plus widget

Widget Overview

The Coordinate widget displays x and y coordinate values on the map. On load of the application, the widget is active and enabled with a pre-configured coordinate system. When a user moves their cursor to different locations on the map, coordinate values change dynamically. Multiple spatial reference systems can be added. There is a set of baseline coordinate systems available for selection to simplify the addition of coordinate systems for commonly used spatial references and enable provision of coordinate system configuration to other widgets.

The coordinate system(s) added to this widget during its configuration will also reflect in the Waahi Find Location widget.

The Coordinate widget has the same configuration options as default Web AppBuilder, see:

https://developers.arcgis.com/web-appbuilder/guide/widget-coordinates.htm

 

Configuration

Setting

Description

Include widget names and versions

This is typically for development use. When true, shows version numbers for widgets currently in the application.

Content

A rich text editor, allows content to be displayed to users of your app.

Notes

A default coordinate system will be detected, using the web maps basemap. For this to work correctly, after your web map is selected, open and save the Coordinate widget setting.


 

5.7       Draw and Measure widget

Widget Overview

GBS has created an extended Draw widget, called the Draw and Measure widget.

The Draw & Measure widget allows the user to draw graphics and text on the map, see all drawings in a list and edit/manage these graphics. Manage area and distance measurements of graphics by choosing to show or hide the measurements and by changing measurement units. The user can be allowed to choose their own measurement units, or the widget can be configured to change measurement units dynamically based on measurement size.  Distance measurements dynamically update in a textbox next to the cursor as line features are drawn, and see measurements for drawn features displayed, such as area and perimeter for polygons.

Included

The product includes the following:

         Drawing modes (i.e. point, line, polyline, freehand polyline, triangle, extent, circle, ellipse, polygon, freehand polygon, buffer, drawing by feature, and text) on the map.

         Editing the shape, colour, and name of graphics added during the session.

         Deleting, moving or rotating graphics added during the session.

         Displaying the length of lines in different units of measurement.

         Displaying the area of a polygon in different units of measurement.

         If configured, the user can change/ define the units of measurement that will be used for displaying the length and area of a graphic.

         If configured, the units for measurements will dynamically update based on the size of the drawing.

5.8       Find Location Plus widget

Widget Overview

The Find Location widget enables users to search for a specific location on a map. The widget provides two search methods for searching locations. Users can search for a location of interest on the map by inputting X and Y coordinates into the fields in the widget and then clicking Locate. Alternatively, they can click on the Identify button, click the mouse cursor on the map and the relative X and Y coordinates will be prepopulated into the widget fields. In this latter Identify example, the coordinates are also overlaid on the map next to the point that was clicked.

The projection systems available in this widget uses what was added for the Coordinate Plus widget.

Configuration

Setting

Description

Use coordinate projection systems

Set this on, to use the projection systems (or spatial references) set in the Coordinate widget. The same systems will be shown in the drop down.

Default: true

Graphic colour

The colour of graphics added to the map displaying coordinates.

Point size

The size of points added to the map.

Font size

The font size of the coordinate label added to the map.

Notes

         Different coordinate systems will require and provide coordinates in different formats. For example, NZTM 2000 uses coordinates in this format: X: 1755313; Y: 5921837 where WGS84 uses coordinates in this format: X:174.742; Y: -36.838.

Widget Overview

The Identify widget is one part of the Interrogate Toolset. It enables users to identify features by clicking them on the map. Feature information at a location is returned and displayed as identifiable layers in the custom Results widget. This display is detailed further in the Results widget description.

In turn, this will then also open the Results widget with attribute information about the highlighted feature. For example, if the user opens this url:  https://demo.gbs.kiwi/waahi?query=Parcels,id=6001111, it is expected that the WAB app opens, it locates the parcel in the Parcels layer where the field "id" is 6001111, and the parcel should be selected on the map using the Identify widget automatically and results opened in Results panel. If the feature layer being queried has IsoLink queries associated with it, the IsoLink queries will also be returned in the Results widget.

5.10       Layer List Plus widget

The Layer List Plus widget allows the user to view all operational layers and their symbols in the web map, individually or grouped under a top-level layer. This also allows the user to turn individual layers on and off depending on their preference. The order in which layers appear in this widget corresponds to the layer order in the map. Change the order of layers in the map by moving the layer up or down in the list. This widget can also display a custom, admin specified description/ metadata of the layer.

 

Configuration

Setting

Description

Show title

Tick this checkbox to show the label Layers above the list in the widget.

Show Basemap

Tick this checkbox to see the basemap used in the map in the list.

Show legend

Tick this checkbox to see symbols for layer that are spatial.

Expand all layers by default

Tick this checkbox to have all layers and group layers in the list expanded on load of the widget.

Choose which actions will be shown on the layer context menu

Each spatial layer in the Layers widget has an action menu, which allows the user to:

         Zoom to

         Adjust the transparency of the layer

         Set Visibility Range of the layer

         Enable or disable the layer popup (when clicking the map)

         Show or hide labels (if labels have been enabled and configured for that layer)

         Move the layer up or down the list

         View the layer records in the attribute table

         Show a layer description, details and Download (ArcGIS Server page)

         Show a customised layer description instead, where the attributes to show can be configured when the checkbox is ticked. Instead of using the service description page, which is a technical page and often unavailable, a summary layer description for the layer can be shown in a popup. The popup content can be configured to show different attributes of the service or portal item details page where relevant.

This is done by clicking the "Add layer description attribute" link. A message box appears where the Source, Attributes and Alias can be selected to show for a layer. The source can be from ArcGIS Server or from Portal. The attributes for a Portal item can be the Summary, Description, Terms of Use, and Owner. The attributes for an ArcGIS Server item can be 'Description', 'Service Description', 'Credits', 'Extent', 'Spatial Reference', 'Author', 'Comments', 'Category', 'Keywords', 'Server Version'. If the portal or Service end point has any of this information, it will be displayed if the user clicks to see Additional Information in the widget next to the relevant layer.


 

Choose which layers will be shown on the list

Layers can be shown or hidden from the list, by default layers will be shown.

Add layer group

Layers can be grouped together under a single top-level layer. Add a group, and add layers by name, comma-separated. In this example the Points and Lines layers will be grouped together.

 

Notes

         Defining an organisations layer groups needs to be done by the WAB Admin on implementation or as part of business-as-usual.

         A layer name listed in the Layers widget will be greyed-out if a user is zoomed out of the layers visibility range. The layers name updates to black, standard text once a user zooms into the layers extent.

         Grouping layers in the Layers widget works for feature layers and map service layers.

5.12       Print widget (standard)

Widget Overview
