2.       Gallery

Overview

The Gallery is a configurable application that displays a collection of applications that have been shared to a users Group Content in Portal. Contents of the Gallery application are interactive and are displayed on a single web page. The left-side panel displays different Group categories. A user can click through the panel to access content associated with different Groups in Portal settings.

The Gallery honours authentication protocols set for Group user content. Portal admin can set privacy settings per application to share their content publicly with everyone, within their organization, or with a specific group. A user must be signed into their ArcGIS Online or Portal account with permissions to access secured applications within a designated group. When signed out, a user is only shown public content shared within a group.

The Gallery application can be configured to use a single URL that can be accessed by desktops, tablets, and smartphones.

Details

This product includes the following requirements:

1.       A user can see the following:

a.       A header that shows:

                                                                              i.            A header title;

                                                                            ii.            Logo (desktop only);

                                                                           iii.            A Sort by drop-down menu (desktop only);

                                                                           iv.            A Search textbox;

                                                                             v.            A Sign In/Sign Out button that updates to show Sign In when a user is signed out and Sign Out when a user is signed into ArcGIS Online or Portal.

b.       A left side panel that contains:

                                                                              i.            The title, Groups;

                                                                            ii.            Selectable Group categories linked to their associated Group and content as configured in ArcGIS Online or Portal.

c.       A content area that features:

                                                                              i.            Application information that is prepopulated from an organisations Portal or ArcGIS Online:

                                                                                                            i.      A title;

                                                                                                          ii.      A thumbnail image;

                                                                                                         iii.      A brief description of content;

                                                                                                         iv.      A clickable information icon to access more information about the item in Portal or ArcGIS Online;

2.       A user can do the following:

a.       Search

i.         Search content by user, title, tags, keywords, description, owner, item type;

ii.       See a set of result(s) after performing a search;

iii.     Clear search text using the X button within the Search textbox.

b.       Sort Content

i.         By title;

ii.       Upload date;

iii.     Popularity based on frequency of visits.

c.       Interact with the Groups side panel:

i.         A user can click through Groups in the side panel to access associated content from the selected Group.

d.       Interact with results display:

i.         Hover their mouse over an application to highlight the item;

e.       Interact with an application item and its content:

i.         Click on the title or thumbnail to open the application in a separate web browser;

ii.       Click the i icon in the thumbnail display to access more information about the item in a separate web browser;

f.        Sign in or out of ArcGIS online or Portal:

i.         Authenticated users can access secure and public content in Groups based on Portal settings;

ii.       Unauthenticated users can access public content in Groups based on Portal settings.

Default Configuration

The Gallery page is configured as follows:

Theme settings

1.       The default configuration settings for the Gallerys colour scheme and text can be sourced from one of two places. By default, the Gallery looks to configuration set in the JSON file (for text and background colours for header, body, and buttons, and logo) and if the file is missing one or more of these items then the Gallery will then look to the Organisations Portal or ArcGIS Online Shared Theme settings.

2.       By default, the JSON code for custom theme settings is intentionally left blank enabling the Gallery theme to inherit directly from Portal settings.

3.       Admin can change settings and values in the JSON configuration file for:

a.       Custom Theme:

                                                                       i.            Colour settings can be changed for the Gallery background, an applications window, an applications caption, caption text, font, header, links, header text, buttons, and button text.

b.       Hide/show items:

                                                                       i.            Text displayed on an applications window to indicate the applications authors name;

                                                                      ii.            Description text for each application;

                                                                    iii.            The main header toolbar in the Gallery page;

                                                                    iv.            Text displayed on an application window indicating application type;

                                                                      v.            A logo or image (header);

                                                                    vi.            The Search toolbar (header);

                                                                  vii.            A Sort by toolbar (header);

                                                                 viii.            The ArcGIS Online URL link (header);

                                                                    ix.            The ArcGIS Online URL link (header);

                                                                      x.            Text displayed for ArcGIS Online, View on ArcGIS Online (header);

                                                                    xi.            The Gallery title (header);

                                                                  xii.            The Portal URL (header);

c.       Sorting:

                                                                       i.            Admin can sort applications in either ascending or descending order based on their title.

d.       Configuration:

                                                                       i.            The title name (header);

                                                                      ii.            The URL location of a logo or image (header);

                                                                    iii.            Placeholder text within the Search textbox (header);

                                                                    iv.            The URL location for ArcGIS Online (header);

                                                                      v.            The amount of applications shown per page;

                                                                    vi.            URL Services for geometries, printing, elevation, and geocoding;

                                                                  vii.            Authentication settings for applications;

                                                                 viii.            Groups displayed in the Gallery (configured by their Group IDs)

                                                                    ix.            Proxy URL;

                                                                      x.            Map units for applications;

                                                                    xi.            Inclusion and placement settings for the Compass, Home, Legend, and Search widgets for applications that utilize these widgets;

 

Excluded

None identified.

Pre-requisites

1.       In the config.js file, admin needs to link the Gallery via a unique ID to a configured group in Portal for ArcGIS to inherit theme settings and access content.

2.       The Gallery page authentication settings can be configured for both Portal and ArcGIS Online (AGOL). The sign in behaviour is dependent on adding an application in Portal and updating its sharing settings (e.g. Everyone) and sharing to Groups. There are two properties in the JSON file that reference these settings:

                                                                       i.            appId: The Gallery page will try to load the app upon initialising of the Gallery page. If the applications configurations are set to Private, the Gallery requires a user to be signed in to access the content;

                                                                      ii.            userAuthAppId: references an application shared with an organisations Portal that allows a sign in link for user authorization in the Gallery Page.

                                                                    iii.            Leaving both properties blank: if both properties are left blank, no sign-in link will be shown to the user.

Notes

1.       The Gallery application is supported on all browsers supported by ArcGIS Online, Portal, and ArcGIS Enterprise. The Gallery is only supported on IE 9 when all content, in the ArcGIS Online, Portal for ArcGIS, or ArcGIS Enterprise group, and the gallery application are shared with the public.

2.       Creating an Application: Applications for use in the Gallery page can be created in Portal under a users or organizations My Content section. Admin can create a new application by either using a template, using Web AppBuilder, or using the operations dashboard.

3.       To inherit a logo set in an organizations Portal settings, the headerImage value must be set to True and the headerImageLocation URL value must be left blank In the JSON configuration file for the Gallery page. If the headerImageLocation URL value is left blank or set to false, no logo will appear in the Gallery page header.

3.       Theme

Overview

The Waahi theme represents the look and feel of the application and is based on the Web AppBuilder Foldable theme style. Content includes a collection of panels, styles, and layouts, and a set of preconfigured widgets. The application will inherit some organisation configuration from the clients Portal settings.

Details

A user can see the following:

1.       Client branding (inherited from the Shared Theme settings in Portal);

2.       A client logo (Inherited from the Organisation settings page in Portal);

3.       An app title (inherited from the web map name in Portal, or the Portal publisher can create an app title in the app template);

4.       A set of widgets (available on load);

5.       A set of on-map navigation tools;

6.       A search toolbar;

7.       A scale bar;

8.       An in-map coordinate display.

Excluded

1.       Change in placement of on-screen widgets;

2.       Change in placement of the widget container.

Pre-requisites

1.       The client has the following brand configuration in Portal. If it is not present, the viewer will use default colours. [In the Shared Theme Settings on Portal]

a.       Header colours;

                                                               i.      Background: Used for the viewer header background colour, widget header background colour and the active state of on-screen widgets (navigation tools). The background colour cannot be set to light as this will clash with the header widget colours (white).

                                                             ii.      Text: Used for the header text colour.

b.       Body colours;

                                                               i.      Background: Not used.

                                                             ii.      Text: Not used.

c.       Button colours;

                                                               i.      Background: Used for button background colour in the widgets. Can be either dark or light coloured as the button perimeter has a border colour which is set to the same colour as the button text.

                                                             ii.      Text: Used for button text colour in the widget.

2.       The client has set their logo in the Organisation settings page in Portal. This must be at least 65px x 65px.

Notes

1.       The header container is a responsive container. The wider the logo and the longer the app title then the less widgets display in the container. When there are too many widgets for the container, the remainder fall into a More widget to the far right.

2.       There are some intricacies with the brand colour inheritance. E.g. Some widgets may have some specificity settings that override the portal inheritance. This isnt something that has been planned but is a by-product of the years of code evolution and releases out of Esri. Our aim is that eventually we will ensure portal inheritance maintains higher priority over various widget code.

 

4.       Widgets

1.1     Legend widget

Widget Overview

The Legend widget displays labels and symbols for layers in the map. It supports dynamic, tiled, image, feature, KML layer types, and WMS layers with an associated legend URL.

Widget Details

A user can see the following:

1.       A Legend widget icon;

2.       The default state of the Legend widget on load of the app is closed;

3.       Map layers in the widget are displayed based on the order they added to the map;

4.       Content in the widget automatically updates if the visibility of a layer or sublayer changes;

5.       The Legend window only shows symbol and label content for layers visible at the current map scale.

 

Default Configuration Details

1.       The Legend content is aligned to the left;

2.       Auto update is enabled;

3.       Respect current map scale is enabled.

Alternative Configuration Details

A WAB Admin can:

1.       Set the Legend widget to display in an alternative alignment;

2.       Set the Legend widget so it does not automatically update if the visibility of a layer or sublayer changes;

3.       Set the map so it does not respect the current map scale;

4.       Change the title of the widget.

Excluded

None identified.

Pre-requisites

None identified.

Notes

None identified.

1.4     Add Data widget

Widget Overview

The Add Data widget enables a user to add data to a map by searching for layers in ArcGIS Online or Portal for ArcGIS content, entering URLs, or uploading local files including shapefiles, CSV, GPX, and GeoJSON. Any graphics added to the map (using the Waahi Draw & Measure widget or the Waahi Locate widget) will appear above any other added layers, preventing overlapping. These graphics layers are temporarily added to (and removed from) the map. Users cannot save layers to the map.

Widget Details

A user can:

1.       See an Add Data widget icon;

2.       See the default state of the Add Data widget on load of the app is closed;

3.       See a Search tab;

a.       Sort by data type;

b.       Sort by data relevance;

c.       See a toggle button enabling searching of layers within or outside the map area;

d.       Search and add data from the users My Content;

e.       Search and add data shared within their organisation;

f.        Search and add data from ArcGIS Online;

g.       Search and add data shared within a (curated) group.

h.       A maximum of 30 items per page for the search result;

i.         Scroll between pages;

j.         View the total layers added;

k.       See an Add function to add the layer to the map. The added layer will appear in the Layer List widget as well;

l.         Where a layer has been added, there is a Remove function to remove the layer from the map. The layer is also removed from the Layer List widget;

m.     A Details function to get information about the layer.

4.       See a URL tab;

a.       Add ArcGIS Server web service/s, WMS OGC web service/s; KML layer/s; GeoRSS feed/s; CSV file/s to the map;

5.       See a File tab;

a.       Upload local files to the map, including shapefiles, CSV, GPX, and GeoJSON;

Default Configuration Details

1.       The number of items per page is set to 30;

2.       Allow My Content is enabled;

3.       Allow My Organisation is enabled;

4.       Allow ArcGIS Online is enabled;

5.       Allow Curated is enabled;

6.       Allow URL is enabled;

7.       Allow File is enabled.

Alternative Configuration Details

A WAB Admin can:

1.       Configure Search settings for data within the following:

a.       Adjust the number of items per page;

b.       Disable/Enable Allow My Content;

c.       Disable/Enable My Organisation;

d.       Disable/Enable ArcGIS Online;

e.       Disable/Enable adding data via a Curated group;

f.        Disable/Enable adding data via a URL;

g.       Disable/Enable adding data via a File.

2.       Configure whether a user can search all layers in the scope or only active map layers;

3.       Change the title of the widget.

Excluded

This functionality is not supported:

1.       These graphics layers are temporarily added to (and removed from) the map;

2.       Users cannot save layers to the map.

Pre-requisites

1.       Enabling adding data through the Allow Curated lets a user search for data shared within a group. This requires the WAB Admin to enter a group ID with the group: prefix as a curated filter.

Notes

1.       The graphics layers added to the map using the LGA Location Solution Draw & Measure widget and the LGA Location Solution Locate widget are prioritised over other layers from the Webmap, preventing overlapping;

2.       Added layers are also added to the Layer List widget;

3.       Removing added layers also removes them from the Layer List widget;

 

 

 


1.5     Layers widget

Widget Overview

The Layers widget provides a list of operational layers, their symbols and allows a user to turn individual layers on and off. Layers with expansion arrows show the symbology associated with the layers features. The order in which layers appear in this widget corresponds to the layer order in the map. This widget provides functionality allowing a user to change the order of layers in the map by moving the layer up or down in the list.

Widget Details

A user can see:

1.       A Layer List widget icon;

2.       The default state of the Layer List widget on load of the app is closed;

3.       Layers that are currently outside their visible scale range are automatically greyed out;

4.       Metadata for every dynamic map service and feature layer;

5.       A check box next to each layer that allows them to control its visibility;

6.       An expansion arrow next to the layers check box to display its symbols;

7.       Layers are displayed in the widget according to the order of the layers in the Webmap;

8.       Grouped sub-layers in group layers.

9.       A Layer Menu icon enabling them to turn all layers on; turn all layers off; expand all layers; collapse all layers;

10.   A Layer Context Menu enabling them to Zoom To; Transparency; Enable/Disable Pop-up; Move up/Move down; Open Attribute Table; Description/Show Item Details/Download;

11.   Any data layers they have added will also be added to the Layer List widget;

12.   Removing added data layers also removes them from the Layer List widget.

Default Configuration Details

 

1.       All configuration options are selected.

Alternative Configuration Details

A WAB Admin can:

1.       Select/deselect the Legend check box which controls whether the user can click the expansion arrow next to the layer to will display its symbols;

2.       Configure the following to show/hide in the layer context menu: Zoom To; Transparency; Enable/Disable Pop-up; Move up/Move down; Open Attribute Table; Description/Show Item Details/Download;

3.       Configure which properties should be displayed in the grouping of sub-layers;

4.       Configure a custom metadata display, including the properties to display for both feature and map service layers;

5.       See instructions under the, Add Layer Group box for grouping multiple layers;

6.       Change the title of the widget.

Excluded

1.       Defining an organisations layer groups needs to be done by the WAB Admin on implementation or as part of business-as-usual.

Pre-requisites

None identified.

Notes

1.       A layer name listed in the Layers widget will be greyed-out if a user is zoomed out of the layers extent. The layers name updates to black, standard text once a user zooms into the layers extent.

2.       Grouping layers in the Layers widget works for feature layers and ArcGISDynamic layers.

1.6     Coordinate widget

Widget Overview

The Coordinate widget displays x and y coordinate values on the map. On load of the application, the widget is active and enabled with a pre-configured coordinate system. When a user moves their cursor to different locations on the map, coordinate values change dynamically. Multiple spatial reference systems can be added. There is a set of baseline coordinate systems available for selection to simplify the addition of coordinate systems for commonly used spatial references and enable provision of coordinate system configuration to other widgets.

Widget Details

A user can:

1.       See on load of the map, as set of coordinates overlaid on the map in the bottom left corner of the map;

2.       See that moving the mouse pointer forces the coordinate values to change dynamically;

3.       Select an alternative coordinate system to the default or the one in current use;

Default Configuration

1.       No coordinate systems are selected or predefined;

2.       The Round coordinate to box is set to 3 decimal places;

3.       The Show thousand separators checkbox is selected.

 

Alternative Configuration Details

 

A WAB Admin can:

1.       Add, edit or delete a set of coordinate systems;

2.       Nominate which coordinate systems a user can select;

3.       Change the title of the widget;

4.       Change the widget icon.

Excluded

None identified.

Pre-requisites

None identified.

Notes

 

1.       The coordinate system displayed on run time is based on that linked to the basemap.

2.       The coordinate widget shares the coordinate systems configured in the WAB Admin to the LGA Local Solution Locate widget.

3.       For the default coordinate system of the Webmap, the coordinates will change dynamically when a user moves the mouse pointer on the map. If the default coordinate system is changed then the user must click the coordinate button again to display the new coordinates (adding a point on the map actions the transformation). GBS has investigated the extension of this functionality to show the coordinates update dynamically irrespective of coordinate system. We suggest the product retain the existing functionality for performance considerations. If the functionality was to be modified, then a transformation would be required on every hover change. This adds system performance complexity as each hover requires a series of web requests.

4.       Esri have announced the release of a new COTS coordinate widget that does transformation on hover change. We will look to the feasibility of incorporating this widget in as part of an upcoming release, in the context of any other streamlined or additional functionality it has.

 

 

1.7     Find Location widget

Widget Overview

The Find Location widget enables users to search for a specific location on a map. The widget provides two search methods for searching locations. Users can search for a location of interest on the map by inputting X and Y coordinates into the fields in the widget and then clicking Locate. Alternatively, they can click on the Identify button, click the mouse cursor on the map and the relative X and Y coordinates will be prepopulated into the widget fields. In this latter Identify example, the coordinates are also overlaid on the map next to the point that was clicked.

Widget Details

A user can:

1.       See a Find Location widget icon;

2.       See the default state of the Find Location widget on load of the app is closed;

3.       See the same set of coordinate systems in the Find Location Projection dropdown list that has been configured and set in the Coordinate widget by the WAB Admin;

4.       Search for the location of interest on the map by inputting X and Y coordinates and then click Locate;

5.       Display the coordinates at a location of interest by clicking on the Identify button, click the mouse cursor on the map and then see the relative X and Y coordinates prepopulated into the fields in the widget window;

6.       Change/switch between coordinate systems.

Default Configuration Details

1.       The coordinate systems are selected (this means the systems configured in the Coordinate widget are used).

2.       The symbology used when the coordinates are highlighted in the Identify use case is pre-set.

Alternative Configuration Details

A WAB Admin can:

1.       Select/deselect the coordinate projection systems. If not selected, this means the admin must add their own coordinate systems, so the ones configured in the Coordinate widget are not used;

2.       The symbology used when the coordinates are highlighted in the Identify use case is pre-set;

3.       Change the title of the widget;

4.       Change the widget icon.

Excluded

None identified.

Pre-requisites

1.       The coordinate systems available for selection in the Find Location widget are based on the those configured in the Coordinate widget.

2.       The client must configure the Coordinate widget to use the Find Location widget, given their dependency.

Notes
